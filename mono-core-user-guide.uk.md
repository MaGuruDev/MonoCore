# MaGuru Monobank Core — Посібник користувача

**Версія:** 1.1.6  
**Сумісність:** Adobe Commerce / Magento Open Source 2.4.4 – 2.4.x  
**PHP:** 8.1 або вище  
**Потребує:** MaGuru Core  
**Ліцензія:** Безкоштовно

---

## Огляд

MaGuru Monobank Core — це **базовий модуль** для всіх інтеграцій MaGuru з Monobank. Він керує з'єднанням з API Monobank і надає спільну інфраструктуру для модулів оплати, розстрочки та фіскалізації.

Сам по собі цей модуль не додає жодних функцій, помітних покупцю. Його потрібно встановити й налаштувати перед будь-яким іншим модулем MaGuru Monobank (Payment, MonoChast, Fiscal).

---

## Що він робить

- Керує з'єднанням з **Monobank Acquiring API** (стандартна обробка платежів)
- Керує з'єднанням з **MonoChast (Частина) API** — окремим API для розстрочки з власною автентифікацією
- **Перевіряє ваші API-креденшали** прямо з адмін-панелі через AJAX-кнопку, яка підтверджує назву мерчанта та ЄДРПОУ
- Показує правильний **Webhook URL** для реєстрації в кабінеті еквайрингу Monobank
- Підтримує **налаштування по кожному сайту (website)** окремо — кожен сайт Magento може мати власні креденшали Monobank
- Забезпечує ECDSA-перевірку підпису вебхуків для безпечної обробки
- Підтримує кілька середовищ для MonoChast: Production, Sandbox і Stage

## Встановлення

### Крок 1 — Встановлення через Composer

```bash
composer require maguru/module-mono-core
bin/magento setup:upgrade
bin/magento setup:di:compile
bin/magento cache:flush
```

> **Примітка:** якщо ваш магазин працює в режимі production, після компіляції також запустіть `bin/magento setup:static-content:deploy`.

### Крок 2 — Перевірка встановлення

```bash
bin/magento module:status MaGuru_MonoCore
```

У виводі має бути `Module is enabled`.

## Налаштування

Перейдіть у **Stores → Configuration → MaGuru → Monobank Integration**.

> **Магазини з кількома сайтами (websites):** усі параметри конфігурації можна встановити окремо для кожного сайту. Перемкніть селектор области (scope) вгорі сторінки конфігурації, щоб задати різні креденшали для різних сайтів.

---

### Секція 1: General (Загальні)

#### Enable Monobank Integration (Увімкнути інтеграцію Monobank)

Вмикає або вимикає інтеграцію з Monobank для обраної области.

- **Yes** — інтеграція активна
- **No** — усі модулі Monobank неактивні; жодних викликів API не робиться

Використовуйте цей перемикач, щоб тимчасово призупинити інтеграцію, не видаляючи нічого.

#### HTTP Request Timeout (s) — Таймаут HTTP-запиту (с)

Максимальний час очікування відповіді від API Monobank.

- **За замовчуванням:** 15 секунд
- Збільшуйте лише якщо стикаєтесь з таймаутами на повільних з'єднаннях

#### HTTP Connect Timeout (s) — Таймаут HTTP-з'єднання (с)

Максимальний час на встановлення початкового з'єднання з Monobank.

- **За замовчуванням:** 5 секунд

#### Debug Logging (Детальне логування)

Коли увімкнено, записує детальні дані запитів і відповідей API у `var/log/mono_core.log`.

- **За замовчуванням:** No
- Вмикайте лише для діагностики — вимикайте на production-магазинах під час звичайної роботи

### Секція 2: Acquiring

> **Цю секцію додає модуль MaGuru Monobank Payment** — вона з'являється лише після встановлення цього модуля. Якщо у вас встановлений тільки Monobank Core, цих полів ви не побачите — це очікувана поведінка, а не помилка.

Ця секція налаштовує з'єднання зі стандартним acquiring API Monobank (використовується Monobank Payment).

#### API Token (X-Token)

Ваш токен API еквайрингу Monobank (зберігається у зашифрованому вигляді).

**Де знайти:**
1. Увійдіть у свій кабінет еквайрингу Monobank
2. Перейдіть у розділ **API**
3. Скопіюйте значення API-токена

#### Validate Token (Перевірити токен)

Натисніть цю кнопку, щоб миттєво перевірити ваш API-токен. У разі успіху модуль покаже назву мерчанта та номер ЄДРПОУ як підтвердження. Якщо бачите помилку — перевірте значення токена ще раз.

> Ця перевірка звертається до API Monobank у реальному часі й потребує, щоб ваш сервер мав вихідний доступ по HTTPS до Monobank.

#### Payment Webhook URL

Поле лише для читання, яке показує URL, на який Monobank має надсилати запит при настанні події з платежем (оплата завершена, скасована тощо).

**Потрібна дія:** скопіюйте цей URL і внесіть його в кабінеті еквайрингу Monobank як webhook-ендпоінт. Без цього ваш магазин не отримуватиме оновлення статусу платежів у реальному часі.

#### API Base URL (Dev Override)

Необов'язкове перевизначення базового URL API Monobank. Залишіть порожнім, щоб використовувати `https://api.monobank.ua`. Встановлюйте це значення лише на локальний mock-URL для розробки/тестування.

### Секція 3: MonoChast (Розстрочка)

> **Цю секцію додає модуль MaGuru MonoChast** — вона з'являється лише після встановлення цього модуля. Якщо MonoChast не встановлений, цих полів ви не побачите — це очікувана поведінка, а не помилка.

Ця секція налаштовує з'єднання з API Monobank Частина (розстрочка). Вона актуальна лише якщо ви плануєте використовувати модуль MaGuru MonoChast.

#### Environment (Середовище)

Обирає, яке середовище MonoChast використовувати:

| Варіант | Коли використовувати |
|---|---|
| **Production** | Робочий магазин, що приймає реальні замовлення з розстрочкою |
| **Sandbox** | Тестування із симульованими транзакціями (без реальних грошей) |
| **Stage** | Стейджинг-середовище Monobank для передпродакшн-тестування |

#### Store ID (Production / Sandbox / Stage)

ID магазину, наданий Monobank для вашого акаунту Частина. Для кожного середовища показується окреме поле: **Store ID (Production)**, **Store ID (Sandbox)**, **Store ID (Stage)**.

#### Store Secret (Production / Sandbox / Stage)

Секретний ключ для HMAC-автентифікації з API MonoChast (зберігається зашифровано). Для кожного середовища показується окреме поле: **Store Secret (Production)**, **Store Secret (Sandbox)**, **Store Secret (Stage)**.

#### Chast Callback URL

Поле лише для читання, яке показує URL, на який Monobank надсилає оновлення статусу замовлень розстрочки. Внесіть цей URL у кабінеті Monobank Частина.

> Store ID, Store Secret та Chast Callback URL показуються окремо для кожного середовища (Production / Sandbox / Stage). Переконайтесь, що заповнили правильні значення для середовища, яке використовуєте.

## Часті запитання

**Q: Де знайти мій API-токен Monobank?**

Увійдіть у кабінет еквайрингу Monobank (наданий Monobank при активації вашого мерчант-акаунту) і перейдіть у розділ налаштувань API. Токен — це довгий рядок, що унікально ідентифікує ваш мерчант-акаунт.

---

**Q: Для чого потрібен Webhook URL?**

Monobank використовує цей URL, щоб надсилати вашому магазину сповіщення про події платежу в реальному часі (успішна оплата, реверс тощо). Без реєстрації webhook URL у кабінеті Monobank ваші замовлення не оновлюватимуться автоматично — довелося б перевіряти статуси платежів вручну.

---

**Q: Чому для Sandbox і Stage середовищ MonoChast окремі креденшали?**

Кожне середовище — це повністю окрема система Monobank з власним доступом до API, і кожна потребує власних креденшалів. Використовуйте Sandbox або Stage для тестування розстрочки без створення реальних фінансових транзакцій. Перемикайтесь на Production лише коли магазин готовий приймати реальні замовлення.

---

**Q: У мого магазину кілька сайтів (websites). Чи може кожен використовувати різні акаунти Monobank?**

Так. Перемкніть область конфігурації на потрібний сайт у випадаючому списку scope і введіть креденшали, специфічні для цього сайту. Кожен сайт працює незалежно.

---

**Q: Кнопка "Validate Token" показує помилку.**

Перевірте наступне:
1. API Token введено правильно (без зайвих пробілів чи символів)
2. Ваш сервер може достукатись до API Monobank по HTTPS (перевірте firewall / правила вихідних з'єднань)
3. API Token активний і не прострочений у вашому кабінеті Monobank

---

**Q: Я не бачу налаштувань "MonoChast". Цей модуль обов'язковий?**

Секція MonoChast з'являється в конфігурації Monobank Integration лише після встановлення окремого модуля MaGuru MonoChast. Якщо ви використовуєте тільки Monobank Payment, цієї секції не буде — це очікувано, MonoChast опціональний.

---

## Підтримка

Перед зверненням у підтримку:

1. Запустіть перевірку Validate Token в адмін-конфігурації та занотуйте результат
2. Увімкніть Debug Logging, відтворіть проблему і надайте відповідні рядки з `var/log/mono_core.log`
3. Зазначте версію Magento, версію PHP та версію MaGuru Monobank Core
